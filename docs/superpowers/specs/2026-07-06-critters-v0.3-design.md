# critters v0.3 — "make it cool" design (2026-07-06, S60 round 2)

Trent-approved scope, all five items plus two riders:

- RIDER 1: every new 3D element (trees, rocks, all of it) uses the SDF blend-shell
  technique. No plain meshes in the world.
- RIDER 2: harness grows a multi-angle visual inspector (move camera, focus subjects,
  shoot from many angles) because visible artifacts remain: occasional clipping
  weirdness, "vertex bounds explosions" (shrapnel spikes), faces set too deep inward
  (creases at prim overlaps, e.g. crimson quad neck). Fix the defaults BEFORE adding
  content. Claude should "see" while working.

## 0. Quality pass (first, blocking)

- `CRITTERS.view(i, theta, dist, h)` debug API: place camera at azimuth theta around
  critter i; `tools/inspect.mjs` shoots N angles x close/far into `verify/inspect/`.
- Root causes identified (index.html VSH main):
  - Inward creases: `bury` smoothstep(0.55, 1.15) sends overlap verts to negative iso
    (-rT*0.4); triangles spanning buried+surface verts cut through the skin. Soften:
    bury later (deeper threshold) and tuck shallower.
  - Vertex explosions: Newton steps `p -= g*(d-iso)` unclamped; bad gradients in smin
    valleys launch verts. Clamp per-step travel to ~1.2*rT.
  - Clipping: fixed boundingSphere 4*S too tight for serpent chain (up to ~3.3 behind
    pos) and long ropes; frustum pops at screen edge. Scale bounds per archetype.
- Verify: before/after inspect sheets on same seeds; featured cast must look identical
  or better from 6 angles.

## A. Meadow (SDF blend-shell flora)

- StaticShell builder: same prim/shader pipeline as Critter but built once; only trees
  get per-frame endpoint writes (canopy sway = slow sine on canopy blob prims + trunk
  lean; no verlet needed). Rocks/mushrooms/flowers fully static (skip per-frame writes).
- Props: tree (trunk round-cone + 2-4 canopy blobs), rock clusters (2-3 gray-green
  blobs), mushroom (stem + cap + spots), flower tuft (thin stem prims + colored head).
- Seeded scatter ring outside the play area (r 6..14), sparse inside (avoid spawn zone);
  deterministic from scene seed; rerolls re-scatter.
- Perf budget: <= 10 props visible, each one draw call pair (body+outline share the
  cached program from the perf fix; uCount handles size). Keep total prims/frame sane.

## B. Owl archetype

- Perched bird: sits on tree canopy or ground stumps; big blinky eyes (reuse eye system,
  large r), head swivel (headYaw sweeps with occasional fast snap, beyond normal clamp),
  tiny wing nub prims. Occasionally glides (flyer-style bank) to another perch.
- Key 7 + UI button. genRecipe branch. Featured "angry owl" easter egg recipe (nod to
  clayafterdark in the thread).

## C. Name-to-critter

- `#n=<text>` URL param + input field inside the credits panel ("make YOUR critter"):
  string -> 32-bit hash (mulberry32 seed) -> genRecipe with deterministic archetype
  pick. Spawns front-and-center with a toast showing the name. Share button includes
  the name when the nearest critter came from one.

## D. Ambient layer

- Procedural audio bed: filtered-noise wind (slow LFO gain) + sparse random birdsong
  chirps (existing synth voice, higher register). Obeys the same mute toggle.
- Cloud shadows: 2-3 large soft dark sprites drifting slowly across the ground
  (reuse ground-patch look) + slightly warmer horizon gradient (simpleclouds lineage).

## E. Ecology behaviors

- Steering additions to wander(): loose flock cohesion (drift toward centroid of
  same-archetype neighbors), flee burst when the possessed critter charges at them,
  curious follow (nearest 1-2 critters trail the possessed player at a distance).
- Babies: 20% recipe roll spawns a 0.45x mini-me that follows its parent (path-follow
  at distance, like serpent segments but a separate critter with the same palette).

## Build order & workers

0 (Fable, blocking) -> A -> B -> C -> D -> E. A-E built by Sonnet workers with tight
specs, sequential (single-file repo), each verified via headless harness (verify.mjs +
inspect.mjs multi-angle) and reviewed by Fable before commit. Push per feature; live
check after each push. Version bumps to v0.3.0 at completion.
