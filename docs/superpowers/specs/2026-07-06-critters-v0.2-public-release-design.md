# critters v0.2 — public release design (2026-07-06)

Take the v0.1 sandbox public at **tront.xyz/critters** as its own repo, with honest
provenance credits, and make it "a bit better": demo-ready on phones, shareable by link,
one new archetype, and more interaction juice. Single-file `index.html` stays; no build
step, no framework.

## 3. Public-demo readiness

- **Touch controls**: tap ground = call critter; one-finger drag = orbit; pinch = zoom;
  tap on a critter = pet (see §6). `touch-action: none` on the canvas; pointer events
  unify mouse + touch.
- **On-screen UI**: small button row — spawn buttons per archetype (5 in phase A;
  serpent's appears with phase C), reroll, share, sound toggle, `?` credits. Styled to match the
  pastel look; auto-fades when idle like the hint pill. Keyboard shortcuts all keep
  working.
- **Loading state**: minimal splash (title + soft spinner) until the first frame renders;
  hides on first rAF. CDN failure shows a friendly "needs internet" message instead of a
  black page.
- **Favicon**: inline SVG/emoji-style data-URI (no extra asset file).
- **Discord/social embed** (must look good when linked):
  - `og:title`, `og:description`, `og:url`, canonical, `og:image` →
    `https://tront.xyz/critters/og-image.png` (absolute URL), `og:image:width/height`
    1200×630, `twitter:card summary_large_image`, pastel `theme-color` for the Discord
    accent bar.
  - `og-image.png`: staged 1200×630 headless screenshot of a framed critter crowd
    (fixed seed, camera framed via `CRITTERS.frame`/manual placement), committed to the
    repo root.
- **GA4**: `G-27SVZGYHZ3` snippet, same as other tront.xyz pages.
- **Perf on phones**: cap `devicePixelRatio` at 2; keep default spawn count modest.

## 4. Share links

- **Scene sharing**: the current seed is always reflected in the URL (`?seed=N`,
  `history.replaceState` on reroll/spawn) so copying the address bar reproduces the
  scene layout deterministically.
- **Critter sharing**: share button (UI row; also works while possessing with a pad)
  encodes the nearest/possessed critter's full recipe as base64 JSON in the hash
  (`#r=...`). Opening such a link spawns that exact critter front-and-center on load,
  plus the normal ambient cast. Copies to clipboard with a "link copied" toast.
- Recipe blobs are small (~15 lines of JSON) so URLs stay well under practical limits.

## 5. More creature variety

- **New archetype: serpent** (key 6, spawn button): legless slitherer. Head steers with
  the existing locomotion root; body segments follow the head's path history
  (path-follow chain), with a lateral slither sine on top. Segments are SDF prims like
  everything else; tongue flick as a tiny rope. Reuses walker steering + rope tech —
  cheapest big-variety win. Featured recipe added; `genRecipe()` learns the archetype.
- **Richer decoration rolls** for all archetypes: horns, head crests, back spikes,
  cheek puffs — each a low-probability extra prim group in `genRecipe()`, wired with
  the existing small-`k` thin-part rules so they don't weld or dissolve.
- **Wider palette space**: a few new palette families (dusty/two-tone/high-contrast)
  while staying pastel-compatible.

## 6. Interaction & juice

- **Petting**: click/tap directly on a critter (raycast against its prim bounds) =
  happy squash-and-stretch pulse + heart puff particles + chirp. Works on possessed
  critters too.
- **Social gaze**: idle critters occasionally pick a nearby critter and look at it for
  a few seconds (reuses the existing eye/head look-target system).
- **Procedural audio**: tiny WebAudio blips — hop launch/land, pet chirp, spawn pop.
  Synthesized (sine/noise envelopes), no assets. Muted until first user gesture
  (autoplay policy); sound toggle in the UI row remembers choice in `localStorage`.

## 7. Verification

Headless Chrome CDP-over-Node pipeline (established pattern):
- Screenshot rounds per phase into `verify/` (new files, keep old ones).
- Touch behaviors via CDP `Input.dispatchTouchEvent` + mobile emulation metrics.
- Share-link round-trip: spawn → encode → reload with hash → screenshot match.
- Serpent gait close-ups; deco rolls sampled across seeds; console-error capture on
  every round; `?seed=` determinism.
- Live-site check after Pages deploy: fetch `https://tront.xyz/critters/` + og-image,
  and validate embed meta (curl the HTML, confirm absolute og:image URL resolves).

## 8. Phasing (ship-first)

- **A — Live**: credits/about panel, touch + UI row, loading state, favicon, embed
  meta + og-image, GA4, DPR cap → push, create repo, enable Pages, verify live,
  games-page card. (Site is public and presentable from here.)
- **B — Share links**: seed-in-URL + recipe hash links + toast.
- **C — Variety**: serpent archetype, decorations, palettes.
- **D — Interaction**: petting, social gaze, procedural audio.

Each phase = separate commit(s) on the public repo, verified before push.

## Out of scope (YAGNI)

- No swimmer/pond, no barcode battler, no creature editor UI, no multiplayer, no
  backend. No code splitting or bundler — one HTML file remains the deliverable.
